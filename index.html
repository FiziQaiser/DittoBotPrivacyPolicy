Ditto Bot - Privacy Policy

By using the Ditto Discord bot, you accept its privacy policy.

1. What does Ditto store?

Nothing permanently. Ditto does not store server data, messages, or user information on any external database.

All data (such as server structure, roles, and recent messages) is processed in-memory only during the execution of a cloning command. Once the operation is complete, the data is discarded immediately.

2. Who can access the data?

No one, including developers, has access to your server's data or messages. All processing is done live in Discord without logging or external storage.

  • No data is stored that could be accessed later by developers or third parties.
  • Ditto only performs actions you explicitly request via commands.

3. How can I ensure my server data is removed?

  • Since Ditto does not store any data persistently, there’s nothing to delete.
  • If you wish to revoke all access, simply kick the bot from your server.

4. Why does Ditto process server data temporarily?

Ditto temporarily accesses server structure and recent messages only to perform the cloning operation requested via the /copy_this_server command.

  • Server data is required to recreate roles, categories, channels, and permissions accurately.
  • Recent messages are optionally copied (if permissions allow) to help replicate the communication context.
  • All of this is done in real-time and discarded immediately after execution.

5. How do I request complete removal or ask for privacy help?

Since Ditto does not store any data, no deletion request is necessary. However, you are always welcome to join our support server if you have concerns or questions.

6. Still have questions?

Feel free to hop into our support server — we're happy to chat and clarify anything over a virtual coffee ☕.
